1.Why are event listeners the preferred way to handle events?

Answer: So that it is easy to read and understand, this is very important for working with code other people wrote, or that you wrote a while ago.

2.What is the difference between good comments and bad comments?

Answer: Good comments are short and descriptive

3.What is an array?

Answer: An array is an object that can hold more than one value.

4.What are arrays useful for?

Answer: They let us create lists of items that we can then iterate through

5.How do you access an array element?

Answer: Using it’s index(starting at 0), eg: 1st item is [0], 2nd is [1] and so on.

6.How do you change an array element?

Answer: By assigning a value to it’s index eg: cars[1] = “volvo”

7.What are some useful array properties?

Answer: Length(), useful for for loops

8.What are some useful array methods?

Answer: .join() to combine, .indexOf() to find things, push(),pop() and splice() to modify items.

9.What are loops useful for?

Answer: Iterating through arrays, doing repetitive tasks, doing something until a condition is met.

10.What is the break statement?

Answer: A statement that lets you jump out of a loop or switch when you want to.

11.What is the continue statement?

Answer: A statement that lets you skip through the current iteration of a loop and onto the next one.

12.What is the DOM?

Answer: The document object model, it is the structure of elements that makes up our html pages.

13.How do you target the nodes you want to work with?

Answer: Use one of the selector methods, eg: document.querySelector(), document.querySelectorAll(), document.getElementByID().

14.How do you create an element in the DOM?

Answer: Document.createElement(tagname)

15.How do you add an element to the DOM?

Answer: Use the method .appendChild() on the parent element

16.How do you remove an element from the DOM?

Answer: Use the method removeChild() on the parent element

17.How can you alter an element in the DOM?

Answer: Get the element using one of the selector methods from earlier, then use .setAttribute(“attribute”,”value”) on it.

18.When adding text to a DOM element, should you use textContent or innerHTML?

Answer: TextContent, since innerHTML might remove existing html structure.

19.Where should you include your JavaScript tag in your HTML file when working with DOM nodes?

Answer: At the bottom of the body tag.

20.How do “events” and “listeners” work?

Answer: An event is any action the user performs, of which there are many (the most common is probably click).
listeners are attached to elements and wait for one of the aforementioned actions to occur to it, they then fire and execute their code.

21.What are three ways to use events in your code?

Answer: To react to clicks, a change in an element, or when the mouse hovers over an element.

22.Why are event listeners the preferred way to handle events?

Answer: Because they let us set up actions for each specific kind of event, for example a button could have a click action, a hover action and an onload (eg enable) action.

23.What are the benefits of using named functions in your listeners?

Answer: They can be called from outside the listener

Answer: queryselector returns the node, querySelectorAll returns a node list.

26.What does a “nodelist” contain?

Answer: A collection of nodes, some properties like length.

27.Explain the difference between “capture” and “bubbling”.

Answer: Capture is the process of the event heading down to find the target element, bubbling is the effects of the event propagating up from the element through to the window.

28.What is the difference between objects and arrays?

Answer: Objects are a wide variety of complex data objects, an array is a specific kind of object that holds a list of items.

29.How do you access object properties?

Answer: Either Object.property or object[“property”]
